# 개선 설명 생성 프롬프트

원본 요구사항과 개선된 요구사항을 비교하여, 지정된 규칙들에 대한 개선 설명을 작성해주세요.

## 입력 정보

**원본 요구사항:**
{original}

**개선된 요구사항:**
{improved}

**개선된 규칙 목록 (점수 변화가 큰 순서):**
{rules}

## 출력 형식

각 규칙에 대해 다음 형식으로 설명을 작성하세요:

**[Rule ID] {규칙명}**
- **개선 내용**: {문제점}을 {해결방법}으로 개선하여 규칙을 만족시켰습니다.

**중요 지침:**
- 점수는 작성하지 마세요 (점수는 별도로 계산됩니다)
- 원본과 개선본을 비교하여 실제로 변경된 내용만 설명하세요
- 가능한 경우 원본과 개선된 텍스트를 인용하세요
- 각 규칙에 대해 간결하고 명확하게 작성하세요
- 원본에 없는 수치, 단위, 시간 값은 임의로 추가하지 마세요.

## 예시

**[P2] Modal Verb**
- **개선 내용**: "~한다"를 "~해야 한다"로 변경하여 의무형 표현으로 개선했습니다.

**[R3] Appropriate Subject-Verb**
- **개선 내용**: 주어가 명확하지 않았던 "시스템은"을 "OO 시스템은"으로 구체화하여 주체를 명확히 했습니다.
